# Purpose

> V1 Alignment Patch: The client API continues to expose Notification resources, preferences, read/acknowledge state, and delivery status. Domain Event subscription is backend-internal and is not exposed as a frontend business subscription API.

The Notification API defines how users, clients, and future AI agents receive, manage, and acknowledge notifications generated by Highlight Signal.

Notifications are delivery objects.

They ensure that important Signals and Recommendations reach the correct recipients through the appropriate communication channels.

The Notification API provides reliable delivery tracking while remaining independent from business decision generation.

---

# Domain Responsibility

The Notification API is responsible for:

* Notification retrieval
* Notification acknowledgement
* Notification dismissal
* Delivery history
* User notification preferences
* Notification scheduling
* Notification search

The Notification API is **not** responsible for:

* Signal generation
* Recommendation generation
* Channel implementation
* Email rendering
* Push notification providers

Delivery execution is handled by backend services.

Notification access requires Workspace membership.

Users may only access notifications addressed to themselves unless they have elevated permissions.

```text id="notifapi03"

member

manager

admin

owner
```

Managers and above may inspect delivery history for operational purposes.

# Delivery Flow

The Notification API delivers business decisions to users and systems.

```text id="notifapi37"
Signal

↓

Recommendation

↓

Notification API

↓

Delivery Service

↓

Communication Channel

↓

User / AI Agent

↓

Acknowledgement
```

The Notification API provides a consistent delivery contract regardless of the underlying communication provider.

---

# Final Rule

The Notification API exposes the delivery layer of Highlight Signal.

Signals determine **what happened**.

Recommendations determine **what should happen**.

Notifications determine **who is informed, through which channel, and whether the message has been successfully received and acknowledged**.

Every notification should be traceable, reliable, and independently auditable across the complete delivery lifecycle.
