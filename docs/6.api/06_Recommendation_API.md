# Purpose

> V1 Alignment Patch: Recommendation endpoints also expose Human Review, formal Decision, and business-level Action transitions within the Recommendation Domain. Insight is analysis content, not an independent V1 API resource.

The Recommendation API defines how clients retrieve, evaluate, execute, and provide feedback on Recommendations generated by Highlight Signal.

A Recommendation is the operational output of the Decision Engine.

It transforms Signals into actionable decisions for users, teams, and future autonomous AI agents.

The Recommendation API is designed to support both human decision-making and automated execution workflows.

---

# Domain Responsibility

The Recommendation API is responsible for:

* Recommendation retrieval
* Recommendation search
* Action retrieval
* Recommendation lifecycle
* User feedback
* Recommendation execution status
* Recommendation timeline

The Recommendation API is **not** responsible for:

* Signal generation
* Evidence collection
* Notification delivery
* Dashboard rendering

Recommendation generation is handled by the AI Recommendation Engine.

# Endpoints

---

## List Recommendations

```text
GET /api/workspaces/{workspace_id}/recommendations
```

Returns paginated Recommendations.

### Query Parameters

```text
page

per_page

status

recommendation_type

priority

expected_impact

signal_id

sort
```

### Example

```text
GET /recommendations?status=pending&priority=high
```

### Response

```json
{
  "data": [
    {
      "id": "recommendation_uuid",
      "title": "Enable Content Security Policy",
      "recommendation_type": "Fix",
      "priority": 95,
      "expected_impact": "high",
      "confidence": 97.6,
      "status": "pending",
      "signal_id": "signal_uuid",
      "created_at": "2026-07-02T09:30:00Z"
    }
  ],
  "meta": {
    "pagination": {
      "page": 1,
      "per_page": 20,
      "total": 58
    }
  }
}
```

# Lifecycle Endpoints

---

## Accept Recommendation

```text
POST /api/workspaces/{workspace_id}/recommendations/{recommendation_id}/accept
```

Marks a Recommendation as accepted.

### Permission

```text
member

manager

admin

owner
```

### Request Body

```json
{
  "comment": "Will implement today."
}
```

### Response

```json
{
  "data": {
    "status": "accepted"
  }
}
```

---

## Start Recommendation

```text
POST /api/workspaces/{workspace_id}/recommendations/{recommendation_id}/start
```

Moves the Recommendation into execution.

### Response

```json
{
  "data": {
    "status": "in_progress"
  }
}
```

---

## Complete Recommendation

```text
POST /api/workspaces/{workspace_id}/recommendations/{recommendation_id}/complete
```

Marks the Recommendation as completed.

### Request Body

```json
{
  "summary": "CSP header deployed successfully."
}
```

### Response

```json
{
  "data": {
    "status": "completed",
    "completed_at": "2026-07-02T10:30:00Z"
  }
}
```

---

## Dismiss Recommendation

```text
POST /api/workspaces/{workspace_id}/recommendations/{recommendation_id}/dismiss
```

Dismisses a Recommendation.


The Recommendation API bridges decision analysis and execution.

```text

↓

↓

```

The Recommendation API is the operational interface of the Decision Engine.

---

# Final Rule

The Recommendation API exposes executable business decisions.

Signals explain **what requires attention**.

Evidence explains **why it matters**.

Recommendations define **what should be done next**.

Every Recommendation should remain explainable, traceable, measurable, and executable, providing a stable contract for users, integrations, and future autonomous AI agents.
