# 01 Signal Concept

Version: v1.0

---

# Purpose

Signal 是 Highlight Signal 最核心的資料單位。

所有分析、決策、建議、通知與 AI 推理，都不是直接建立在網站、SEO、GA、Ads 或任何單一資料來源上，而是建立在 Signal（訊號）之上。

Highlight Signal 的使命不是收集更多資料，而是找出真正值得被關注的訊號。

---

# Definition

Signal 是一個具有決策價值的資訊單位。

一筆資料只有在能夠改變使用者的理解、判斷或行動時，才會被視為 Signal。

Signal 並不等同於 Data。

Data 是原始資訊。

Information 是整理後的資訊。

Signal 則是經過分析後，具有行動價值的資訊。

Decision 是根據多個 Signal 所做出的判斷。

因此：

Data → Information → Signal → Decision → Action

---

# Signal Philosophy

Highlight Signal 相信：

不是所有資料都有價值。

真正重要的是能夠幫助使用者快速理解現況、降低判斷成本並採取正確行動的訊號。

AI 的工作不是產生更多內容，而是發現更多真正重要的 Signal。

---

# Signal Lifecycle

每一個 Signal 都會經過完整生命週期。

Source

↓

Collect

↓

Analyze

↓

Evidence

↓

Confidence

↓

Recommendation

↓

Decision

↓

Action

↓

Feedback

Signal 並不是一次性的分析結果。

它會隨著新的資料持續更新、提升可信度，甚至失效。

---

# Signal Characteristics

一個有效的 Signal 應具備以下特性：

- 可驗證（Verifiable）
- 可追蹤（Traceable）
- 可解釋（Explainable）
- 可比較（Comparable）
- 可量化（Measurable）
- 可行動（Actionable）

如果一項資訊無法支持後續決策，就不應被視為 Signal。

---

# Signal Sources

Signal 可以來自不同來源。

例如：

- Website
- SEO
- AEO
- GEO
- Google Analytics
- Google Search Console
- Google Ads
- Social Media
- Security Scan
- Cloud Infrastructure
- Business Metrics
- CRM
- User Behavior
- AI Observation

Highlight Signal 不限制訊號來源。

任何具有決策價值的資訊，都可以形成 Signal。

---

# Signal Categories

Signal 可以依照不同面向分類。

Business Signal

影響商業目標的訊號。

Marketing Signal

影響曝光、流量、轉換的訊號。

Security Signal

影響網站與雲端安全的訊號。

Technical Signal

影響系統效能與可靠性的訊號。

Behavior Signal

使用者行為所形成的訊號。

AI Signal

由 AI 推論、歸納或預測所產生的訊號。

---

# Signal Structure

每一個 Signal 都應包含完整描述。

- ID
- Title
- Description
- Category
- Source
- Timestamp
- Severity
- Confidence
- Evidence
- Recommendation
- Status
- Related Signals

Signal 並不是單純的文字。

它是一個可被 AI 理解、引用、關聯與推理的知識物件。

---

# Signal Relationship

Signal 不應孤立存在。

不同 Signal 之間可能存在：

- Cause
- Effect
- Correlation
- Dependency
- Sequence
- Conflict

當多個 Signal 建立關聯後，可以形成更高層級的洞察。

---

# Signal Priority

Signal 應依據重要程度排序。

Priority 會綜合考量：

- Severity
- Business Impact
- Confidence
- Time Sensitivity
- User Context

因此，同一個 Signal 對不同 Workspace 或不同角色，優先順序可能不同。

---

# AI Perspective

AI 不直接回答問題。

AI 的第一步，是辨識有哪些 Signal。

第二步，是驗證哪些 Signal 值得相信。

第三步，是建立 Signal 之間的關聯。

最後才形成 Recommendation 與 Decision。

因此：

Signal 是 AI 推理的最小單位。

不是 Prompt。

不是 Document。

不是 Report。

而是 Signal。

---

# Future Evolution

未來 Signal 將逐步演化為更完整的知識模型。

包括：

- Signal Graph
- Signal Timeline
- Signal History
- Signal Cluster
- Signal Pattern
- Signal Memory
- Signal Prediction

Signal 將成為 Highlight Signal AI Decision Engine 的核心基礎。

---

# Summary

Highlight Signal 並不是一套資料分析工具。

它是一套以 Signal 為核心的 Decision Intelligence Platform。

所有資料最終都會轉化為可理解、可驗證、可追蹤、可行動的 Signal。

Signal 是 Highlight Signal 的最小決策單位，也是 AI 建立洞察、建議與決策能力的核心基石。
